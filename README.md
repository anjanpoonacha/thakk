# Thakk — Kodava Takk Language Model

A QLoRA fine-tuned language model for Kodava takk that transliterates Kodava words and phrases into Marathi Devanagari script with phoneme-level pronunciation hints, confidence flags, and grammar trap warnings. The goal is to help Marathi speakers read and pronounce Kodava words correctly without prior exposure to the language — each output includes sound-by-sound breakdowns using familiar English and Marathi analogies, uncertainty flags for phonemes that diverge from Marathi expectations, and red-flag warnings for common grammatical traps such as retroflex consonants, geminate stops, and tonal length distinctions.

## Use Case

Marathi speakers learning Kodava takk can input any Kodava word or phrase and receive a Devanagari rendering alongside a structured pronunciation guide. The flag system signals when a sound requires extra attention versus when it maps cleanly to a known Marathi phoneme.

## Quick Start

**Prerequisites**

- Python 3.10+
- Apple Silicon Mac (tested on M3 Pro, 18GB unified memory)
- `pip install mlx-lm`

**Train**

```bash
bash train.sh
```

Training takes approximately 1–2 hours on M3 Pro. The adapter is saved to `adapters/kodava/`.

**Test**

```bash
bash test.sh
```

Runs 5 verification prompts against the fine-tuned adapter and prints structured output for manual review.

## Flag System

| Flag | Meaning |
|------|---------|
| ✅ | High confidence — sound maps cleanly to a known Marathi/Hindi phoneme |
| ⚠️ | Uncertain — sound exists in Kodava but has no direct Marathi equivalent; treat with caution |
| 🔴 | Grammar trap — structural rule that Marathi speakers commonly misapply (e.g. geminate consonants, retroflex distinctions) |
| 🟡 | Provisional — sourced from textbook only; not yet verified by a native speaker |

## Data Sources and Truth Hierarchy

This project uses a two-tier truth system:

1. **`elementary_kodava_FINAL.md`** — the base knowledge layer. A complete Kodava takk textbook covering phonology, grammar, vocabulary, and sentence structure. Used as the primary source for training data generation.

2. **`kodava_corrections.md`** — the native speaker override layer. Contains corrections from a native Kodava speaker that supersede any conflicting information in the textbook. When generating training examples or verifying outputs, entries in this file always take precedence over the textbook.

The override layer exists because published textbooks occasionally contain romanization inconsistencies or outdated usages. Native speaker corrections capture how the language is actually spoken.

## Adapter Reusability

The QLoRA adapter (`adapters/kodava/`) is saved separately from the base model (`mlx-community/Qwen2.5-7B-Instruct-4bit`). This means:

- The base model (~4GB) is downloaded once and never modified.
- The adapter (~50–100MB) encodes all Kodava-specific knowledge.
- To run inference after training, load the base model and attach the adapter — no retraining required.
- The adapter can be shared independently and applied to any compatible Qwen2.5 7B 4-bit deployment.

## Adding New Corrections

The intended feedback loop is:

1. The model outputs a transliteration flagged with ⚠️ (uncertain) or 🟡 (textbook-only, unverified).
2. A native speaker reviews the output and provides the correct pronunciation or form.
3. The correction is added to `kodava_corrections.md` with a brief note on what the textbook got wrong.
4. `bash train.sh` regenerates training data incorporating the new correction and retrains the adapter.

This loop allows the model to improve incrementally without rebuilding the dataset from scratch. The corrections file is the primary artifact to maintain over time.
